¿QUÉ ES GIT

Es el mas popular sistema de control de versiones
que como su nombre lo indica controla los cambios que hagas al proyecto
optimo para trabajos en equipo.

Antes de empezar debemos aprender unos comandos basicos para usar Git
En git hay 3 estados el primero workin directory que es donde trabajamos con nuestros archivos
staging area que es una parte intermedia donde preparamos los archivos que deseamos guardar
y el tercer estado un repositorio que es donde estara guardado nuestros archivos

    • git init: le decimos al sistema que usaremos git en el proyecto
      git add nombredelarchivo:con esto subimos los archivos del working directory al staging area
      git status:nos dice en que estado estan los archivos si el working directory,status o repository
      git commit:es cuando ya estamos seguros de los archivos que deseamos subir,y subimos los archivos al repositorio tomando algo como una primera toma del programa
      git push:con este lo subimos a un repositorio a un servidor como la primera version
      git pull:Si estamos trabajando con mas desarrolladores nos trae los cambios que estos han hecho al proyecto fuente
      git clone:descarga una copia desde el servidor al pc.

    • Instalacion de Git: Buscamos en el navegador la pagina de git y damos click en download y listo :V es un exe se instala damos click.
    Al dar el 5 siguiente debemos cambiar la segunda opcion por la tercera
    para poder usarlo desde el cmd(terminal de windows)

    Primer ejemplo de git

    creamos una carpeta, y abrimos una terminal con windows+r,
    escribimos cmd y damos enter.algunos comandos de la terminal.
    ls=muestra los archivos en la carpeta donde estamos.
    pwd=muestra el directorio actual.
    cd carpeta=nos movemos a la carpeta que especifiquemos luego del cd
    nos movemos a la carpeta que usaremos como proyecto y tipeamos git init y ya nuestro proyecto usara git.
    Debemos inicialmente configurar git con nuestro correo y con un usuario para saber
    que desarrollador es el que esta haciendo los cambios.
    git config --global user.email "tuemail" git config --global user.name "tunombre"
    luego veremos que archivos y en que estado de los 3 estamos con el comando git status.
    donde como apenas comenzamos nos dira que no hay archivos subidos.
    pasemos un archivo del working directory al staging area con el comando git add nombre del archivo
    si damos otro git status nos mostrara el archivo en verde y que esta esperando ser confirmado para subirse
    a continuacion crearemos nuestro primer snapshot o foto o punto de partida de nuestro proyecto
    tipeamos git commit -m "un pequeño comentario del cambio "
    y ya tendriamos la primera "foto" con nuestros cambios.
    con git log vemos el registro de commits que hemos hecho y el mensaje que anteriormente escribimos al hacer el commit.
    nos dira un id del commit que acabamos de hacer y el autor y fecha que lo hizos
    si usamos git chekout y realizamos un cambio a algun archivo de los subidos git restaurara los cambios a la version del commit
    con git diff podemos ver las diferencias del archivo modificado y el del commit
    para agregar esos cambios realizados al archivo debemos nuevamente pasar del working area
    al staging area con git add luego con git commit -m " " y listo
